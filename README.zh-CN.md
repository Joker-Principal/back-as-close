# back as close

这是一个 Chrome 浏览器扩展，当你在浏览历史记录的起始位置（即无法继续后退时）触发后退操作时，会自动关闭当前标签页。

## 工作原理

扩展会拦截后退导航尝试，并判断是否还有历史记录可以后退：

1. 当你通过鼠标侧键、Alt+Left 或 BrowserBack 键触发后退时，扩展会调用 `history.back()`。
2. 扩展等待 200ms，让导航操作完成，然后检查 URL 是否改变。
3. **如果 URL 改变了**：说明进行了正常的导航，扩展不做任何处理。
4. **如果 URL 没有改变**：说明你已在第一个页面（没有更多历史记录了），扩展会关闭该标签页。

这种方式比检查 `history.length` 更加简单可靠，因为 `history.length` 可能受到同源重定向等边界情况的影响。

## 支持的触发方式

- **鼠标侧键** — 通过 `auxclick` 事件捕获
- **Alt+Left 键盘快捷键** — 通过 `keydown` 事件捕获
- **BrowserBack 键** — 当浏览器发出 BrowserBack 键事件时捕获

## 不支持的方式

- 浏览器工具栏的后退按钮（Chrome 扩展无法拦截）
- `data:`、`blob:` 或 `about:` 协议的页面（内容脚本无法在这些页面运行）

## 安装步骤

1. 克隆此仓库或下载源代码。
2. 打开 Chrome，进入 `chrome://extensions/`。
3. 启用**开发者模式**（右上角的切换开关）。
4. 点击**加载已解压的扩展程序**，选择本项目目录。

扩展加载完成后，会立即在所有页面上开始拦截后退操作。

## UI 界面

- 扩展不提供弹出式菜单或工具栏操作界面。
- 安装后，扩展只通过注册的内容脚本和后台服务工作线程运行。

## 调试日志

调试日志默认启用，可提供关于扩展行为的详细信息：

- **内容脚本日志** 在页面的 DevTools 控制台中显示，前缀为 `[BackAsClose][Content]`。
  - 查看方式：在任意页面右键 → **检查** → **控制台** 标签页。
  - 日志内容：捕获的后退事件、URL 比较、关闭决策和页面卸载事件。

- **后台服务工作线程日志** 在扩展的服务工作线程控制台中显示，前缀为 `[BackAsClose][Background]`。
  - 查看方式：打开 `chrome://extensions/` → 找到本扩展 → 点击**检查**下的**Service worker**。
  - 日志内容：标签页关闭操作和遇到的任何错误。

如要禁用日志，请在 `src/history-check.js` 和 `src/service-worker.js` 中将 `DEBUG_LOG_ENABLED` 设置为 `false`。

## 手动验证

使用以下简单场景测试扩展：

1. 打开新标签页，依次浏览：**页面 A** → **页面 B** → **页面 C**。
2. 使用鼠标侧键或按 Alt+Left 触发后退。
3. 验证：现在你在**页面 B** 上。再次触发后退。
4. 验证：现在你在**页面 A** 上。这证明了正常的后退导航功能正常工作。
5. 在**页面 A** 上再次触发后退。
6. 预期结果：**标签页关闭**（因为没有更多历史记录可后退）。

你也可以打开 DevTools 控制台（按 F12）来查看来自 `[BackAsClose][Content]` 的详细日志，了解每一个决策过程。

## 实现细节

- **基于 URL 的检测**：扩展通过比较 `history.back()` 前后的 URL 来判断是否发生了导航，而不是检查 `history.length`。这更加可靠。

- **无状态设计**：扩展不保存持久化状态、会话存储或跨标签页通信数据。每个标签页独立运行。

- **平台兼容性**：支持任何通过 http、https 或 file 协议提供的页面。会跳过可编辑元素（INPUT、TEXTAREA、SELECT、contentEditable），避免干扰文本编辑。

- **标签页隔离**：当标签页关闭时，`beforeunload` 事件会设置一个标志，防止后台脚本接收陈旧消息，从而确保没有跨标签页的副作用。

- **轻量级**：整个扩展仅约 200 行代码，零外部依赖。
